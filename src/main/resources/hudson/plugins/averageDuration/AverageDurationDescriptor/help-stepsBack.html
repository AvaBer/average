Set the number of steps back in the build history the scope will be to find candidates (Healthy builds).
Higher numbers may and probably will increase load time.

Set this to a value around a point where the target candidate amount would most likely to reach its goal.

If left blank or if a value that is less than 1 is used the default value will be used aslong as the requirement is met.
Requirement: Equal to or Greater than Candidates
Default: 6
Min Value: 1
Max Value: 50
